# Kirchhoff IR Fix：ir.json → .tex + .png + 布局诊断

**一句话**：给一个 ir.json（或 circuit.tex），自动修常见问题 + 检测线条布局违规 + 序列化渲染。

**与主 skill 的关系**：主 skill（kirchhoff-eye）是从图片出发的全流程。本 skill 是从 IR 出发的快捷通道，也是**布局修复铁律**的执行入口。

## 使用方法

```bash
# IR 修复+渲染（原有功能）
python scripts/ir_fix_and_render.py <ir.json> [-o <out_dir>] [--dpi 300]

# 布局检查（新增）
python scripts/ir_fix_and_render.py <circuit.tex> --layout-check [--json]
```

## 自动修复能力

| 错误码 | 问题 | 修复方式 |
|--------|------|----------|
| E008 | 四路汇合点未声明 junction | 自动添加 junction |
| E008 | 十字交叉未声明 crossing | 自动添加 plain crossing |

其他错误（E001-E007, E013-E014）标记为「需人工」。

# 线条布局铁律（--layout-check）

> 来源：`out/circuit2/` 四轮实战修复（2026-07-12），每轮暴露的问题反哺为铁律。

## 铁律总表

| # | 铁律 | 检测方法 | 修复策略 |
|---|------|----------|----------|
| **零** | **布局修复优先于网格吸附** | — | 修复要求离网坐标时照做，网格吸附让步 |
| 一 | **只有水平和竖直导线，绝对禁止斜线** | 每段 Δx≠0 且 Δy≠0 → 斜线 | 连接点向引脚平移 / 增加 L 形 elbow |
| 二 | **不能有折返线（水平+竖直）** | 连续同向段方向相反（dx1·dx2<0） | 移动元件使两端引脚对齐 |
| 三 | **电流源中点对齐** | 接入点 ≠ 两管引脚几何中点 | 移动电流源+N_点+轨线+标签 |
| 四 | **移动元件消除线头，级联所有受影响点** | 每次修改后全量验证 | 逐级移动直到整链正交 |
| 五 | **验证五步法：正交+回溯+中点+引脚+级联** | 五项全部检查 | 不允许整图扫一眼 |
| 六 | **修改优先级：先拓扑后外观** | — | 错连>缺件>错型>错位>错标签>错样式 |

### 铁律零：布局修复优先于网格吸附

**当布局修复要求坐标不落在 0.5 网格时，优先执行修复，忽略网格吸附规则。**
- 例：电流源中点 x=1.75（离网）→ 照放 1.75，不强制移动晶体管使中点落网。
- 例：为消除斜线将 N_4 从 y=2.5→2.55（离网）→ 照改，E003 网格警告自动豁免。
- 唯一约束：离网坐标只能是**修复的副作用**，不能主动创造离网设计。

## 铁律一：绝对禁止斜线

**判定**：对于每一段导线 segment(p1, p2)，若 `abs(dx) > EPS` 且 `abs(dy) > EPS`，则为**斜线**。

**常见根因**：三极管/MOS 等多端件引脚偏移（如 npn C=(0, +0.55)）不落在 0.5 网格，与 0.5 网格上的连接点产生微小斜线（典型值 0.05 单位）。

**修复二选一**：
1. **平移连接点**：将 junction/坐标移至引脚同轴（如 N_4 从 y=2.5→2.55）。
   - 前提：其他接入该点的导线均同 x 或同 y，移动不产生新斜线。
2. **增加 L 形 elbow**：引脚→同轴离网 waypoint→主走线；每一段都必须严格水平/竖直。
   - 场景：连接的两端引脚 y 不同，无法用一个点同时对齐。

> 注：按铁律零，平移连接点到引脚同轴时若产生离网坐标（如 2.55），
> **布局修复优先**——照改，网格吸附自动豁免。

## 铁律二：禁止折返线

### 水平折返
连续两段均为水平（dy=0），且 dx1·dx2 < 0（先右后左，或先左后右）。

**典型 case**：导线 4→4.5→4.4，越过 Q4.B(x=4.4) 走到 N_8(x=4.5) 再折返。线头 = (4.4, y) 到 (4.5, y) 的重复段。

### 竖直折返
连续两段均为竖直（dx=0），且 dy1·dy2 < 0（先上后下，或先下后上）。

**更隐蔽的形式**：引脚与总线不同 y，走线先竖直再水平。若另一端引脚可移至同 y，则这段竖直段是"折返"。
- 例：Q3.E 在 y=-0.1，总线在 y=0，走线 Q3.E→(4,0)→Q4.B。若 Q4 下移 0.1 使 Q4.B 也在 y=-0.1，则 W8 变为纯水平直连。

**修复**：移动一端元件使两引脚对齐到同一水平（竖直）线，实现纯水平（竖直）直连。

### 检测算法（回溯检查）

```
for 每条 multi-segment wire path:
  segments = 逐段展开 waypoints
  for i in range(len(segments)-1):
    s1 = segments[i]; s2 = segments[i+1]
    if s1 和 s2 终点=起点（连续）:
      if 同水平 and dx1 * dx2 < 0 → 水平折返
      if 同竖直 and dy1 * dy2 < 0 → 竖直折返
```

## 铁律三：电流源中点对齐

**判定**：若电流源的一个端子接入两晶体管引脚之间的连接导线，接入点必须在两引脚连线的**几何中点**。

**公式**（水平情况）：x_mid = (x1 + x2) / 2。

> 按铁律零：若中点不落 0.5 网格（如 1.75），**照放离网坐标**，不强制移动晶体管。
> 中点对齐 > 网格吸附。

**级联移动清单**：
1. 电流源本体坐标（放到中点）
2. 连接点 N_x 坐标（放到中点）
3. 轨线（Vcc/GND bus）相关段坐标
4. 电流源的 +/- 标签坐标

**旋转情况**：竖直电流源接两管竖直连线，中点在 y 轴，同理。

## 铁律四：级联移动

```
对移动后的点 P：
  for 每条以 P 为端点的导线 W:
    for W 的每个相邻段:
      if 该段变得不水平/不竖直:
        继续移动该段的另一端点，直到恢复正交
```

**停止条件**：整条链路全部正交，或到达外部端口（ocirc 终端）。

## 铁律五：验证五步法

| 步骤 | 检查内容 | 通过标准 |
|------|----------|----------|
| 1. 正交检查 | 每段导线 Δx=0 或 Δy=0 | 0 段斜线 |
| 2. 回溯检查 | 连续同向段的方向符号 | 0 次方向反转 |
| 3. 中点检查 | 电流源接入点位置 | 100% 对齐几何中点 |
| 4. 引脚检查 | 所有多端件引脚 | 100% 接入 |
| 5. 级联检查 | 每处修改的连带导线 | 全部正交 |

**核心教训**：逐段正交 ≠ 整体无折返。步骤 1 通过不能保证步骤 2 通过。

## 铁律六：修改优先级

```
错连（net 接错） > 缺件/多件 > 错型（type 不对） > 错位（位置不准） > 错标签 > 错样式
```

同优先级内每轮只修 **≤5 条**（小步防震荡）。

## 实战案例（来自 out/circuit2/ 四轮修复）

| 轮次 | 发现 | 类型 | 修复 |
|------|------|------|------|
| R1 | 6 处 0.05 单位斜线（Q1.E→N_4 等） | 铁律一 | 平移 junctions + 增加 elbows |
| R2 | I2 接入点 x=1.5 ≠ 中点 x=1.75（离网）→ 照放 1.75，铁律零豁免网格 | 铁律三·中点对齐 | I2+N_4+Vcc轨+标签整体右移 0.25 |
| R3 | W8 走 4→4.5→4.4 水平折返 | 铁律二·水平 | 去掉 N_8 绕路，直连 Q4.B |
| R4 | Q3.E 在 y=-0.1，总线 y=0，竖直折返 | 铁律二·竖直 | Q4 下移 0.1，W8 变纯水平 |

**反哺**：R1 漏检 6 处斜线 → 建立铁律一。R3 漏检 N_8 折返 → 建立铁律二+回溯检测。R4 漏检竖直折返 → 建立"逐段正交≠整体无折返"教训。
